# Integration Patterns

<!-- Overview -->
The workspace uses a simple integration model where each project plays a clear role in the overall financial flow.

<!-- Patterns -->
- COBOL batch programs validate, reconcile and report.
- DB2 stores master data, movements, staging and audit records.
- Java APIs expose account and movement information for online inquiry use cases.

<!-- Purpose -->
The idea is to demonstrate how legacy batch systems and modern APIs can coexist in the same financial environment.
