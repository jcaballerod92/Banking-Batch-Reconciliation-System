# Batch vs API

<!-- Batch -->
Batch processes work with scheduled file-based execution, sequential processing and end-of-run reporting. They are ideal for validation, reconciliation and large-volume financial jobs.

<!-- API -->
APIs provide on-demand access through HTTP requests and return data immediately. They are better for account inquiries, movement lookup and lightweight online services.

<!-- Comparison -->
This portfolio includes both styles because financial systems often combine batch operations for back-office processing and APIs for operational access.
