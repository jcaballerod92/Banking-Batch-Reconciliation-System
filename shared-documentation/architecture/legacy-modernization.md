# Legacy Modernization

<!-- Context -->
A common evolution in banking systems is the transition from mainframe batch processing toward service-based Java platforms.

<!-- Approach -->
This workspace mirrors that evolution by pairing:
- COBOL and JCL for classic batch flow
- DB2 for transactional persistence
- Java APIs for online access and integration

<!-- Value -->
The goal is not to replace legacy technologies, but to show how they can be understood, maintained and modernized in a controlled way.
